## タイポグラフィ

> **原則：対比 (Contrast)**
> 見出しと本文のサイズ・ウェイトに明確な差（ジャンプ率）をつけることで、情報の階層を構築し、スキャナビリティを向上させます。

### フォント

| 用途           | フォント                                         |
| -------------- | ------------------------------------------------ |
| 本文・UI全般   | Montserrat（欧文）+ Noto Sans JP（和文）         |
| フォールバック | Yu Gothic → -apple-system → Hiragino Kaku Gothic |

和欧混植構成。ブラウザは Montserrat でラテン文字を描画し、グリフのない日本語は Noto Sans JP にフォールバックする。フォント最適化として `font-feature-settings: "palt"` を本文に適用。

### テキストサイズ

| 役割                       | 値の例                       |
| -------------------------- | ---------------------------- |
| ページ・セクション見出し大 | `text-[32px] sm:text-[22px]` |
| セクション見出し           | `text-[24px] sm:text-[18px]` |
| カード・ブロックタイトル   | `text-[20px] sm:text-[16px]` |
| 本文                       | `text-[16px] sm:text-[14px]` |
| サブテキスト・ラベル       | `text-[14px] sm:text-[12px]` |
| メタデータ・タグ・日付     | `text-[12px] sm:text-[10px]` |
| 最小補助テキスト           | `text-[10px]`                |

### ルール
- **強弱の対比:** 見出しは `font-bold`（700）以上を使い、本文（`font-normal`）との差を強調する。
- **読みやすさの確保:** 行間（line-height）は本文 `1.8`、見出し `1.32〜1.48` とし、適切な余白を反復させる。
- `whitespace-nowrap` を使い、重要なラベルが改行されないよう保護する。

## カラー

> **原則：反復 (Repetition)**
> 定義されたパレットを繰り返し使用することで、ブランドの一貫性を担保し、特定の色が特定の機能（例：青＝アクション）を指し示すシグナルとして機能させます。

### パレット

| トークン          | 値        | 用途                                 |
| ----------------- | --------- | ------------------------------------ |
| Primary Blue      | `#0079C9` | リンク・アクション・ロゴ・アクセント |
| Body Text         | `#333`    | 本文テキスト                         |
| Secondary Text    | `#666`    | サブテキスト・ボーダー               |
| Muted Text        | `#999`    | 日付・メタデータ・非アクティブ       |
| Background        | `#FAFAFA` | ライトグレー背景                     |
| Surface           | `#F6F6F6` | リスト・バッジ・タグ背景             |
| Border Light      | `#eee`    | セパレーター・薄いボーダー           |
| Border Medium     | `#ccc`    | 標準ボーダー                         |
| White             | `#FFF`    | メイン背景・ヘッダー等               |
| Alert Red         | `#E81F00` | エラー・マーカーテキスト             |
| Prompt Background | `#EBF7FF` | インフォ・コールアウトボックス       |

### ルール
- **一貫性の反復:** Tailwind のデフォルト色は使わず、定義された任意値（`text-[#333]` 等）を**反復**使用する。
- **階層の構築:** 本文（`#333`）と補助テキスト（`#666` / `#999`）の明度差による**対比**で視覚的階層を作る。
- Primary Blue はアクセントに限定し、装飾目的で多用しない。

## スペーシング

> **原則：近接 (Proximity) & 整列 (Alignment)**
> 関連する要素を近づけ、異なるグループを離すことで、情報の関係性を直感的に伝えます。すべての要素は8pxグリッドに基づき整然と配置（整列）させます。

### 基本単位
スペーシングは **8px 基準** の倍数系。

| 用途               | デスクトップ | SP（sm:）    |
| ------------------ | ------------ | ------------ |
| セクション間       | `gap-[64px]` | `gap-[40px]` |
| カード内コンテンツ | `gap-[12px]` | `gap-[8px]`  |
| アイコンとテキスト | `gap-[8px]`  | `gap-[6px]`  |
| 記事内 h2 上余白   | `mt-[120px]` | `mt-[60px]`  |
| 記事内段落上余白   | `mt-[16px]`  | `mt-[12px]`  |

### レイアウト幅（整列の基準）
